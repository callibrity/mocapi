# MocapiServerProperties as record + defaults file

## What to build

Convert `MocapiServerProperties` to a record and move default
values to a `mocapi-server-defaults.properties` file, matching
the pattern used by substrate and other auto-configured modules.

### Properties file

Create `mocapi-server/src/main/resources/mocapi-server-defaults.properties`:

```properties
mocapi.server-name=mocapi
mocapi.server-title=Callibrity Mocapi MCP Server
mocapi.server-version=unknown
mocapi.session-timeout=PT1H
mocapi.allowed-origins=localhost,127.0.0.1,[::1]
```

### Record

Convert `MocapiServerProperties` to a record (if not already):

```java
@ConfigurationProperties(prefix = "mocapi")
public record MocapiServerProperties(
    String serverName,
    String serverTitle,
    String serverVersion,
    String instructions,
    Duration sessionTimeout,
    List<String> allowedOrigins,
    String sessionEncryptionMasterKey) {}
```

No `@DefaultValue` annotations — defaults come from the
properties file. The record is pure shape.

### Auto-config

Reference the defaults file:

```java
@AutoConfiguration
@EnableConfigurationProperties(MocapiServerProperties.class)
@PropertySource("classpath:mocapi-server-defaults.properties")
public class MocapiServerAutoConfiguration {
```

### What about sessionEncryptionMasterKey?

This has no default — it's either provided explicitly or
generated by the `EphemeralMasterKeyEnvironmentPostProcessor`
in `mocapi-example-autoconfigure`. Do NOT add a default for it
in the properties file. If it's missing and the post-processor
isn't on the classpath, the app fails to start with a clear
error.

## Acceptance criteria

- [ ] `mocapi-server-defaults.properties` exists with default
      values for `serverName`, `serverTitle`, `serverVersion`,
      `sessionTimeout`, `allowedOrigins`.
- [ ] `MocapiServerProperties` is a record.
- [ ] Auto-config references the defaults file via
      `@PropertySource`.
- [ ] No `@DefaultValue` annotations on the record.
- [ ] No default for `sessionEncryptionMasterKey`.
- [ ] Users can override any property via `application.yml`.
- [ ] `mvn verify` passes.
