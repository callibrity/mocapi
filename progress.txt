NOTES: Raw string choose/chooseMany customizer overloads were omitted because Java's type inference creates irreconcilable ambiguity between chooseMany(String, List<T>, Function<T,String>) and chooseMany(String, List<String>, Consumer<MultiChoiceConstraints<String>>) when T=String. Users needing customization with raw strings should use the arbitrary objects API with Function.identity() as valueFn. Similarly, enum choose customizer overloads require extracted typed Consumer variables when calling with lambdas due to javac overload resolution limitations with generic methods (choose(name, Class<E>, E) vs choose(name, Class<E>, Consumer<ChoiceConstraints<E>>)).
