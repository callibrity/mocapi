NOTES: Discovered and worked around a bug in methodical-autoconfigure 0.2.0: Jackson3AutoConfiguration's @ConditionalOnBean(ObjectMapper.class) fails because it runs before Spring Boot's JacksonAutoConfiguration creates the ObjectMapper bean. This means Jackson3ParameterResolver is never registered, breaking all JSON-RPC parameter extraction (tools/call gets name=null). Fixed in CompatApplication by explicitly registering the bean. This same bug likely affects mocapi-example — HelloToolIT and Rot13ToolIT are not run by failsafe and would fail if they were. The root fix belongs in the methodical library: Jackson3AutoConfiguration needs @AutoConfiguration(after = JacksonAutoConfiguration.class).
