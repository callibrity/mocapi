WHAT_WAS_DONE: Added new `mocapi-audit` module (`com.callibrity.mocapi.audit`) with `AuditLoggingInterceptor`, `AuditCallerIdentityProvider`, `DefaultAuditCallerIdentityProvider`, `AuditFieldKeys`, `MocapiAuditProperties`. The interceptor emits one SLF4J INFO event on the `mocapi.audit` logger per handler invocation with caller/session_id/handler_kind/handler_name/outcome/duration_ms (always present; `session_id` may be null) and optionally `arguments_hash` (opt-in via `mocapi.audit.hash-arguments=true`, SHA-256 over key-sorted canonical JSON via `Hashes.sha256Of(byte[])` from spec 195) and `error_class` (on failure). Outcomes: `success` default, `forbidden` for `JsonRpcException` code -32003, `invalid_params` for -32602, `error` for anything else -- `CallToolResult.isError=true` still counts as `success`. `safeCaller` falls back to `"anonymous"` if the provider throws or returns null/empty. `sortKeys` recursively canonicalizes JsonNode trees so the arguments_hash is stable across key order. Added `MocapiAuditAutoConfiguration` in `mocapi-autoconfigure` with four `@Order(200)` customizer beans (tool/prompt/resource/resource-template) and two nested conditional configs that bind the default `AuditCallerIdentityProvider`: Spring-Security-backed when `SecurityContextHolder` is on the classpath, trivial `() -> "anonymous"` otherwise. Added `@Order(100)` to all four existing MDC customizer beans in `MocapiLoggingAutoConfiguration` and `@Order(300)` to all four existing observation customizer beans in `MocapiO11yAutoConfiguration` per the spec's chain-ordering contract. Registered `MocapiAuditAutoConfiguration` in `AutoConfiguration.imports`, added `mocapi-audit` to parent pom `<modules>`, `mocapi-bom/pom.xml` dependencyManagement, and `mocapi-autoconfigure/pom.xml` as `<optional>true</optional>`.
